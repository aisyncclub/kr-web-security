# 사례: AI 자동화 → 관리자 페이지 봇 탐지

## 요약
AI 에이전트가 외부 SaaS 관리자 페이지를 자동화 접근하여 봇/인젝션으로 탐지된 사례.

## 발생 경위
1. curl로 로그인 API POST 요청 → reCAPTCHA 차단
2. 브라우저 자동화로 상품 리스트 페이지 빠른 연속 이동
3. JavaScript `fetch()`로 JSON API 엔드포인트 직접 호출 시도
4. URL 파라미터 조작으로 검색/정렬 시도
5. **상대방 보안 시스템이 해당 IP를 봇/인젝션으로 탐지**

## 원인 분석
- 관리자 페이지 자동화 접근이 봇 패턴으로 인식
- 짧은 시간 내 다수 API 엔드포인트 호출 (사람이 불가능한 속도)
- reCAPTCHA 우회 시도

## OWASP 매핑
- A05:2021 Security Misconfiguration (봇 방어 정상 작동)
- A07:2021 Authentication Failures (인증 우회 시도)

## 교훈
```
1. 봇 탐지가 있는 외부 서비스에 프로그래매틱 접근 금지
2. reCAPTCHA 우회 시도 금지
3. 관리자 페이지 API 엔드포인트 직접 호출 금지
4. 패턴: 파일 생성(로컬) → 업로드(사용자 직접)
```

## 부가 사례: AI 요약 오류
같은 날 AI 이메일 요약에서 비밀번호가 잘못 표시됨 (실제 4자리 → AI 요약 5자리).
AI 생성 요약의 수치/지시는 반드시 원본 확인 후 사용.
